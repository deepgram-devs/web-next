# {frontmatter.title}
Deepgram tracks requests made to Deepgram services and associates usage information with Projects.

<ColumnsForCode>
	<div slot="left">

	### Get All Requests

	Generates a list of requests sent to the Deepgram API for the specified project over a given time range. Uses pagination to limit the number of results returned.

	#### Path Parameters

	<div class="feature">
		<header class="notranslate">
			<strong>project_id:</strong> uuid
		</header>
		<main>
						<p>Unique identifier of the project for which you want to retrieve requests.</p>
		</main>
	</div>

	#### Query Parameters

	<div class="feature">
		<header class="notranslate">
			<strong>start:</strong> string
		</header>
		<main>
			<p>Start date of the requested date range. Format is YYYY-MM-DD. Defaults to the time of your first
						request.</p>
					<p>If no time zone is specified, defaults to Coordinated Universal Time (UTC).</p>
		</main>
	</div>
	<div class="feature">
		<header class="notranslate">
			<strong>end:</strong> string
		</header>
		<main>
			<p>End date of the requested date range. Format is YYYY-MM-DD. Defaults to the current time.</p>
			<p>If no time zone is specified, defaults to Coordinated Universal Time (UTC).</p>
		</main>
	</div>
	<div class="feature">
		<header class="notranslate">
			<strong>limit:</strong> integer
		</header>
		<main>
						<p>Number of results to return per page.</p>
						<p class="notranslate"><strong>Default:</strong> 10</p>
		</main>
	</div>
	<div class="feature">
		<header class="notranslate">
			<strong>status:</strong> string
		</header>
		<main>
					<p>Status of requests to return. Enables you to filter requests depending on whether they have succeeded or failed. If not specified, returns requests with all statuses.</p>
					<p class="notranslate"><strong>Default:</strong> null</p>
					<p><strong>Possible values:</strong> <code>null</code>, <code>succeeded</code> OR <code>failed</code></p>
		</main>
	</div>

				</td>
			</tr>
		</tbody>
	</table>
